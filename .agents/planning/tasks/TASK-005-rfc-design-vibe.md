# TASK-005: RFC - "Vibe" e Identidade Visual

**Épico Pai:** [EPIC-002](../epics/EPIC-002-design-definitions.md)
**Status:** `[TODO]`

## Contexto
O aplicativo precisa de uma identidade visual ("Vibe") antes de estilizarmos qualquer tela. O aplicativo do artesão será focado na gestão rápida via mobile, mas também uso denso via desktop (tablets/pc) para coisas administrativas. O usuário migrará de planilhas de gestão, e queremos ser tanto agradáveis quanto altamente eficientes. Precisamos discutir e votar (RFC) as opções visuais antes de bater o martelo (ADR).

## O Que Deve Ser Feito (Missão para o Agente Especialista de UX/UI)
1. Iniciar um documento RFC em `doc/rfcs/RFC-002-design-vibe.md`.
2. Propor no mínimo três "Vibes" de Design distintas para o App:
   - Uma com visual limpo/Notion/Utilitário.
   - Uma acolhedora/Etsy/Artesanal.
   - Outra Moderna/Premium.
3. Para cada Vibe proposta, criar e hospedar mockups rápidos (HTML puro + Tailwind via CDN) na pasta `doc/mockups/` ou linká-los para testar o feeling no browser.
4. Conduzir a revisão dos "comentários" simulando uma discussão até chegar a um consenso ou colher feedback explícito da pessoa de produto (Usuário).

## Critérios de Aceite
- [ ] Documento RFC-002 completo e com os feedbacks documentados.
- [ ] Protótipos funcionais simples criados para exemplificar.
- [ ] Ao final (em task futura), a decisão se tornará num ADR de Design efetivo.
